Cards

Cards combine imagery, a short description, and a link or call-to-action.

I've avoided styling the content of the card, for now. I'd prefer card content to be driven by standard building blocks, like the rest of the DS.

Cards themselves have no explicit size. They flow into their container. This makes them useful in many different scenarios.

With controlled image size

The --image-min-size CSS variable can be used to control the relative size of the image within the overall card. Both percentages and static values may be used. Note that this is a minimum image size. The image will still grow to fit available space.

--image-min-size defaults to 25%.

--image-min-size is applied based on the direction of the card. If the image is on top, then --image-min-size is applied to the height of the image. If the image is on the side, then direction of the card. If the image is on top, then --image-min-size applies to the width.
